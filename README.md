# DayGlass Usage

DayGlass Usage는 Windows 데스크탑에서 하루 동안 어떤 앱과 페이지를 사용했는지 기록하는 개인용 사용시간 트래커입니다. 하루 기준은 자정이 아니라 오전 6시입니다. 즉, 새벽 6시 전까지는 전날 기록으로 이어집니다.

앱의 목표는 단순한 실행 시간 합산이 아니라, 출근부터 퇴근까지의 흐름 안에서 실제 컴퓨터 사용시간, 일한 시간, 딴짓한 시간, 자리비움 시간을 구분해서 보는 것입니다.

## 주요 개념

### 날짜 기준

- 하루는 오전 6시에 시작합니다.
- 예를 들어 2026-05-13 02:00 기록은 26-05-12 날짜에 포함됩니다.
- 화면에는 `YY-MM-DD (요일)` 형식으로 날짜가 표시됩니다.

### 출근과 퇴근

- 메뉴의 `출근` 버튼을 누르면 그날의 근무가 시작됩니다.
- 출근 후에는 버튼이 `퇴근`으로 바뀝니다.
- 퇴근을 누르면 그 시점 이후 사용시간 기록이 멈춥니다.
- 퇴근 후 같은 날 다시 버튼을 누르면 `퇴근취소`로 처리되어 기록이 이어집니다.
- 퇴근취소 시에는 퇴근했던 시점부터 지금까지를 자리비움으로 기록할 수 있는 패널이 뜹니다.
- 오전 6시가 지나 새 날짜가 되면 다시 `출근` 상태로 시작합니다.

## 상단 시간 카드

상단에는 네 가지 시간이 표시됩니다.

- `근무시간`: 출근부터 퇴근까지의 전체 시간입니다. 퇴근 전이면 출근부터 현재까지 계산됩니다.
- `사용시간`: 출근 후 퇴근 전까지 실제 컴퓨터 사용으로 인정된 시간입니다.
- `일`: 실행목록에서 `일`로 분류된 항목들의 사용시간 합계입니다.
- `딴짓`: 실행목록에서 `딴짓`으로 분류된 항목들의 사용시간 합계입니다.

메뉴에서 `초 보기/초 숨기기`를 눌러 상단 네 카드의 초 표시를 켜고 끌 수 있습니다. 앱별 실행목록 시간은 분 단위로 표시됩니다.

## 실행목록

실행목록은 기록된 앱과 페이지를 시간순 또는 사용량순으로 보여줍니다.

- 기본 정렬은 `많이사용`입니다.
- `최신` 버튼을 누르면 최근 사용한 항목이 위로 옵니다.
- 현재 사용 중인 항목에는 `지금` 표시가 붙습니다.
- 각 항목 오른쪽의 `일/딴짓` 버튼으로 분류를 바꿀 수 있습니다.
- KakaoTalk 계열 창은 하나의 항목으로 합쳐집니다.

## 브라우저와 영상 사이트

Chrome, Edge, Brave, Firefox, Opera, Vivaldi 같은 브라우저는 창 제목을 기반으로 현재 페이지를 기록합니다.

YouTube, CHZZK, SOOP 같은 영상 사이트는 일반 브라우저 사용과 별도로 기록됩니다. 영상 사이트가 활성 창이면 마우스나 키보드 입력이 없어도 영상 시청 시간으로 인정됩니다.

PiP 창은 실행목록에는 별도로 중복 기록됩니다. 단, 총 사용시간에는 중복으로 더하지 않습니다.

## 자리비움 기록

자리비움은 컴퓨터를 쓰지 않았지만 이유를 기록하고 싶은 시간을 남기는 기능입니다.

자리비움 패널이 뜨는 경우는 네 가지입니다.

- 출근 후 퇴근 전 상태일 때만 자리비움 기록이 가능합니다.
- 메뉴에서 `자리비움`을 누르면 그 순간부터 시작 시간이 잡힙니다.
- 사용자가 직접 자리비움을 누르지 않았더라도, 시스템이 일정 시간 입력 없음으로 판단하면 그 시점부터 시작 시간이 잡힙니다.
- 퇴근 후 같은 날 `퇴근취소`를 누르면 퇴근했던 시점부터 시작 시간이 잡힙니다.

자리비움 패널에서 내용을 입력하고 `기록`을 누르면, 시작 시점부터 기록 버튼을 누른 순간까지의 시간이 실행목록에 추가됩니다. `취소`를 누르면 기록하지 않습니다.

기본 선택지는 다음과 같습니다.

- 식사
- 미팅
- 수업
- 공부
- 영어
- 화장실
- 수다
- 기타

선택지에서 고를 수도 있고 직접 입력할 수도 있습니다. 자리비움도 `일/딴짓`으로 분류할 수 있습니다.

## 로그

메뉴의 `Log`에서 날짜별 기록을 볼 수 있습니다.

- 날짜를 누르면 날짜 선택기가 열립니다.
- 해당 날짜의 근무시간, 사용시간, 일, 딴짓 시간이 표시됩니다.
- 앱 기록은 같은 이름이면 한 줄로 합쳐 표시됩니다.
- `일/딴짓` 표시는 합쳐진 시간 중 더 많이 차지한 분류를 기준으로 표시됩니다.

## 출퇴근시간 달력

메뉴의 `출퇴근시간`에서 월간 달력 형태로 출근/퇴근 기록을 볼 수 있습니다.

- 파란색 `출`은 출근 시간입니다.
- 살구색 `퇴`는 퇴근 시간입니다.
- `<`, `>` 버튼으로 이전달과 다음달을 이동합니다.

## 메모

메뉴의 `Memo`에서 간단한 할 일을 적을 수 있습니다.

- 메모는 날짜별로 나뉘지 않습니다.
- 입력하면 자동 저장됩니다.
- Clear로 내용을 비울 수 있습니다.

## UI와 창 동작

- 글래스모피즘 느낌의 반투명 UI를 사용합니다.
- 창은 항상 위에 고정됩니다.
- `S/M/L` 크기 전환이 가능합니다.
- `S` 크기에서는 실행목록을 숨기고 핵심 시간과 컨트롤만 보여줍니다.
- 투명도 슬라이더로 창 투명도를 조절할 수 있습니다.
- 앱 아이콘은 사용자가 제공한 3번 로고 이미지를 사용합니다.

## 데이터 저장

기록은 Electron의 `userData` 위치에 `usage-state.json`으로 저장됩니다. 앱 프로젝트 폴더와 기록 저장 폴더는 다릅니다.

프로젝트 폴더:

```text
C:\Users\nuri0\Documents\Codex\2026-05-12\6-yy-mm-nn-ui-ui
```

현재 개발 실행 기준 기록 저장 위치는 보통 다음 계열입니다.

```text
%APPDATA%\DayGlass Usage\usage-state.json
```

Electron 개발 환경에서는 앱 이름과 실행 방식에 따라 `userData` 경로가 달라질 수 있습니다.

## 실행 방법

프로젝트 폴더에서 다음 명령을 실행합니다.

```powershell
npm start
```

## 추후 개발 후보: Google Drive 동기화

이 앱은 개인용 또는 최대 5명 정도의 소규모 사용을 전제로 하면, 대형 서버를 붙이는 것보다 Google Drive 기반 동기화가 가장 가볍습니다.

권장 방향은 다음과 같습니다.

- 비로그인 상태에서는 지금처럼 로컬 기록만 사용합니다.
- 로그인하면 Google Drive의 앱 전용 폴더에 기록 파일을 저장합니다.
- 각 컴퓨터는 로컬 기록과 Drive 기록을 비교해 날짜별로 병합합니다.
- 같은 날짜의 앱 기록은 앱 키 기준으로 합산합니다.
- 출근/퇴근 시간은 더 이른 출근, 더 늦은 퇴근을 기준으로 병합할 수 있습니다.
- 충돌을 줄이기 위해 자동 동기화와 수동 동기화를 둘 다 제공하는 것이 좋습니다.

자동 동기화만 있으면 편하지만, 컴퓨터 여러 대가 동시에 켜져 있을 때 기록 충돌이 생길 수 있습니다. 따라서 적합한 범위는 다음과 같습니다.

- 앱 시작 시 자동 내려받기
- 일정 간격 자동 업로드
- 메뉴에서 `지금 동기화` 수동 실행
- 충돌이 있으면 날짜 단위로 병합

Google 계정 로그인을 직접 붙이는 방식은 OAuth 토큰 저장, 권한 관리, 충돌 처리까지 필요합니다. 구현은 가능하지만 앱 복잡도가 올라갑니다. 개인용이면 먼저 Google Drive 파일 동기화 방식으로 시작하는 것이 가장 현실적입니다.
